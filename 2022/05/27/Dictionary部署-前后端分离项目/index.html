linux每运行一个程序就会产生一个进程，那么可以通过查看nginx进程来判断它是否运行。
ps -ef | grep nginx
nginx监听的是80端口，那么就可以查看80端口运行的程序来判断Nginx是否运行
netstat -anp | grep 80

nginx相关

Nginx (engine x) 是一个高性能的HTTP和反向代理web服务器，其特点是占有内存少，并发能力强。
Nginx提供的负载均衡策略有2种：内置策略和扩展策略。内置策略为轮询，加权轮询，Ip hash。
动静分离，在我们的软件开发中，有些请求是需要后台处理的，有些请求是不需要经过后台处理的（如：css、html、jpg、js等等文件），这些不需要经过后台处理的文件称为静态文件。让动态网站里的动态网页根据一定规则把不变的资源和经常变的资源区分开来，动静资源做好了拆分以后，我们就可以根据静态资源的特点将其做缓存操作。提高资源响应的速度。

npm run serve 是本地环境运行，会在本地用node.js生成一个web服务从而访问页面
npm run build 是打包生成静态文件，通常我们把静态文件上传到服务器后会通过nginx或者tomcat启动一个后台服务器的web服务然后通过配置让该服务指向静态文件达到访问页面的目的
两者之前的区别其实就是web服务生成个启动方式，例如本地运行的时候可以从代码里配置代理来解决跨域问题但是打包以后放服务器上以后代理就不生效了，因为本地代码的代理配置只是对本地启用的node服务进行的配置，如果线上需要做代理就必须在页面部署所用的服务中再进行配置。关于开发环境与生产环境其实只是项目中的一个默认全局变量，默认是开发环境只是执行打包的时候脚手架把默认值修改成了生产环境
